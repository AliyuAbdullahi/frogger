                                                                Konstantin Breu • 2 years ago

                                                                There is a problem of this implementation - and of other similar implementations in the internet. If multiple javascript based animations, which are running at the same time, call the native requestAnimationFrame implementaion (each of them), their callback can be called in the same time (direct subsequently). With the fallback implementation (using setTimeout) this is not the case, causing sync problems. The fallback implementation could check whether there are different open calls. Then it's possible to call the callbacks with one setTimeout. The strategy might depend on the duration of the single callbacks, of course. If they take too long, then they should not be called all together.

                                                                /**
                                                                * RequestAnimationFrame polyfill, based on version of Paul Irish
                                                                * http://paulirish.com/2011/requ...
                                                                *
                                                                * Extended fallback implementation: enable subsequent calls to
                                                                * requestAnimationFrame to be performed in on step (with one setTimeout call)
                                                                * This improves parallel, synchronous animations
                                                                */
                                                                (function() {
                                                                var lastTime = 0;
                                                                var vendors = ['ms', 'moz', 'webkit', 'o'];
                                                                var callbacks = [];
                                                                var canceled = {};
                                                                for (var x = 0; x 0) {
                                                                callbacks.push(callbackObj);
                                                                return callbackObj.id;
                                                                }

                                                                var currTime = new Date().getTime();
                                                                var timeToCall = Math.max(0, 16 - (currTime - lastTime));
                                                                callbacks.push(callbackObj);
                                                                var id = window.setTimeout(function() {
                                                                var callbacksCopy = callbacks;
                                                                callbacks = [];
                                                                for (var i = 0; i < callbacksCopy.length; i++) {
                                                                if (canceled[callbacksCopy[i].id]) {
                                                                continue;
                                                                }
                                                                callbacksCopy[i].callback(currTime + timeToCall);
                                                                }
                                                                canceled = {};
                                                                }, timeToCall);
                                                                lastTime = currTime + timeToCall;
                                                                return callbackObj.id;
                                                                };
                                                                }

                                                                if (!window.cancelAnimationFrame) {
                                                                window.cancelAnimationFrame = function(id) {
                                                                canceled[id] = true;
                                                                };
                                                                }
                                                                }());

                                                                                                                                                      This may not be relevant to anyone but I wanted to share a slight rewrite of the raf pollyfill that doesn't overwrite the window methods but saves them to a separate object:

                                                                                                                                                      var animFrame = (function(req, can) {
                                                                                                                                                      var af = {},
                                                                                                                                                      lastTime = 0,
                                                                                                                                                      vendors = ["webkit", "moz", "o", "ms"];
                                                                                                                                                      af[req] = window.requestAnimationFrame;
                                                                                                                                                      af[can] = window.cancelAnimationFrame;
                                                                                                                                                      for(var x = 0, l = vendors.length; x < l && !af[req]; ++x) {
                                                                                                                                                      af[req] = window[vendors[x]+&#039RequestAnimationFrame&#039];
                                                                                                                                                      af[can] = window[vendors[x]+"CancelAnimationFrame"] ||
                                                                                                                                                      window[vendors[x]+"CancelRequestAnimationFrame"];
                                                                                                                                                      }
                                                                                                                                                      if (!af[req]) {
                                                                                                                                                      af[req] = function(callback, element) {
                                                                                                                                                      var currTime = Date.now(),
                                                                                                                                                      timeToCall = Math.max(0, 16 - (currTime - lastTime)),
                                                                                                                                                      id = window.setTimeout(function() {
                                                                                                                                                      callback(currTime + timeToCall);
                                                                                                                                                      }, timeToCall);
                                                                                                                                                      lastTime = currTime + timeToCall;
                                                                                                                                                      return id;
                                                                                                                                                      };
                                                                                                                                                      }
                                                                                                                                                      if (!af[can]) {
                                                                                                                                                      af[can] = function(id) {
                                                                                                                                                      clearTimeout(id);
                                                                                                                                                      };
                                                                                                                                                      }
                                                                                                                                                      return af;
                                                                                                                                                      }("request", "cancel"));

                                                                                                                                                              Mark Rejhon • 2 years ago

                                                                                                                                                              Another convenient optimization that web vendors have begun to do with requestAnimFrame() is to make them synchronized to VSYNC. "Yes" means browsers that can synchronize requestAnimFrame() to VSYNC:

                                                                                                                                                              IE10: Yes (only at 60Hz)
                                                                                                                                                              Chrome: Yes on all platforms (up to 240 Hz on PC)
                                                                                                                                                              Safari: Yes on Mac, Yes for iOS6, no on PC
                                                                                                                                                              FireFox: No
                                                                                                                                                              Opera: No

                                                                                                                                                              We need to set up a W3C standard for querying the browser for whether we're synchronizing to VSYNC, and what the display refresh is, because it is in many cases /very/ desirable to let requestAnimFrame run at 120Hz if it's a computer system configured to 120Hz refresh. (Battery-operated machines won't be using 120Hz for the foreseeable future, anyway) Animations are much smoother at 120Hz on very fast systems.

                                                                                                                                                              This really needs to be standardized because there are some situations where the application needs to know we're synchronizing to VSYNC, and to be able to go beyond 60Hz.

                                                                                                                                                              (Note: "60" is actually a number that comes from the common refresh rate. I think it stays in the spirit to let it match refresh rate, whenever refresh rate is not 60. Games that use it, should be adaptive anyway -- that is the best practices, and they should not assume that requestAnimFrame should always be 60.)
